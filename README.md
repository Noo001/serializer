## 📋 Описание задачи

**Исходные данные:**
- Множество (массив, порядок не важен) целых чисел в диапазоне **от 1 до 300**
- Количество чисел: **от 5 до 1000**
- Простейшая сериализация: строка вида `"1,300,237,188"`

**Требование:**
> Напишите функцию сериализации / десериализации в строку, чтобы итоговая строка была компактной.  
> **Итоговая строка должна быть в 2 раза короче простейшей сериализации** для любых возможных наборов чисел.

**Дополнительные условия:**
- Сериализованная строка должна содержать только **ASCII символы**
- Можно использовать любой язык программирования
- Нужно приложить набор тестов с коэффициентом сжатия

## 🧠 Решение: алгоритм сжатия

### Идея

Числа находятся в диапазоне 1..300, что меньше 2⁹ = 512.  
Значит, **каждое число можно хранить в 9 битах**.

### Этапы сериализации
```
Исходные числа: [1, 300, 237, 188]
↓
Вычитаем 1 (диапазон 0..299): [0, 299, 236, 187]
↓
9 бит на число: 000000000 100101011 011101100 010111011
↓
Упаковка в байты (big-endian): [0, 37, 86, 187, 215, 187]
↓
Base64 кодирование: "ACVWu9e7"
↓
Добавляем префикс с количеством чисел (4 hex-символа): "0004ACVWu9e7"
```


### Коэффициент сжатия

| Количество чисел | Простая строка (симв.) | Сжатая строка (симв.) | Коэффициент |
|-----------------|------------------------|----------------------|-------------|
| 50              | ~174                   | ~72                  | **2.42x**   |
| 100             | ~350                   | ~142                 | **2.46x**   |
| 500             | ~1750                  | ~704                 | **2.49x**   |
| 1000            | ~3500                  | ~1407                | **2.49x**   |

✅ Для больших наборов чисел коэффициент стабильно превышает **2.0x**

## 🛠️ Технические детали

### Формат сжатой строки
```text
[4 hex-символа: количество чисел] + [Base64 от упакованных байтов]
```


Пример: `0004ACVWu9e7`
- `0004` = 4 числа
- `ACVWu9e7` = Base64 от 6 байтов (4 × 9 бит = 36 бит = 4.5 байта → округлено до 6 байтов)

### Почему Base64?

- ✅ Встроен в браузер (`btoa` / `atob`) — не нужно писать свою реализацию
- ✅ Даёт достаточный коэффициент сжатия (∼2.5x)
- ✅ Строка состоит только из ASCII символов

### Обработка граничных случаев

- Пустой массив → возвращается пустая строка
- Числа вне диапазона 1..300 → выбрасывается ошибка
- Некорректный ввод при десериализации → выбрасывается ошибка

## 🔧 API
serialize(numbers)

Параметры:

    numbers — массив чисел от 1 до 300

Возвращает:

    Строка ASCII — сжатое представление

Пример:
```javascript
serialize([1, 300, 237, 188]) // "0004ACVWu9e7"
deserialize(str)
```
Параметры:

    str — строка, полученная из serialize()

Возвращает:

    Массив чисел (порядок может отличаться от исходного)

Пример:
```javascript
deserialize("0004ACVWu9e7") // [1, 300, 237, 188]
```
## Демонстрация

Интерфейс позволяет:

- Ввести числа вручную (через запятую или пробел)
